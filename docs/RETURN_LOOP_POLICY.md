# Return Loop Policy

Status: approved for pilot planning

Purpose
-------
This document defines the first return policy for AllDone.

It closes `P2.3`.

## Policy summary

The pilot uses:

- deposit-backed returns
- scheduled pickup through the recurring service cycle
- one-cycle grace for normal household delays

## Default rules

- each active household pays a refundable `EUR 40` deposit
- containers are expected back by the next scheduled cycle
- empties should be handed back during delivery or scheduled pickup
- one missed cycle is allowed without penalty if the household remains responsive

## Loss and non-return handling

If containers are not returned:

1. first missed cycle: reminder
2. second missed cycle: account flagged for follow-up
3. repeated non-return: apply deposit-backed loss handling or pause service until resolved

## Breakage handling

Normal accidental breakage should not be framed punitively during pilot one.

Default policy:

- isolated accidental breakage is logged
- repeated abnormal loss or damage triggers operator review
- deposit drawdown is reserved for persistent non-return or clearly excessive loss patterns

## Why deposit-backed returns are the default

- cleaner incentive alignment
- easier household explanation
- better container-pool control
- better pilot evidence on real return behavior

## Why subscription-only trust is not enough for pilot one

A no-deposit model may be possible later, but for the first pilot it would weaken the evidence on:

- true return compliance
- loss economics
- household behavior under real obligations
