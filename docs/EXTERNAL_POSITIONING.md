# External Positioning

Status: active

Purpose
-------
This document defines how AllDone should be framed externally.

The project currently has three distinct states:

- public exposure
- funding readiness
- pilot readiness

They are not the same thing and should not be blurred together.

## Recommended framing now

AllDone should currently be framed as:

- publicly exposed
- funding-ready in narrative structure
- not yet pilot-ready in operational proof

That is the most credible and defensible positioning.

## 1. Public exposure

This is the frame for:

- the GitHub repository
- public readers
- collaborators
- general discovery

The correct claim is:

AllDone is a public concept, research, prototype, and IP-claim repository for a closed-loop home provisioning system.

What this frame should communicate:

- what the system is
- what value it provides
- why it matters
- what exists already
- what still needs to be proven

What this frame should not imply:

- that the business is launch-ready
- that the pilot already works
- that the economics are already validated

## 2. Funding readiness

This is the frame for:

- angel conversations
- pre-seed conversations
- strategic partners
- early serious diligence

The correct claim is:

AllDone has a coherent system thesis, a clear pilot model, a clear value proposition, and a structured proof plan suitable for early-stage funding discussion.

What this frame should communicate:

- the problem and alternatives
- the reason the system matters
- the first pilot wedge
- the key risks
- what capital would fund
- what proof the round is intended to buy

What this frame should not imply:

- that the risks are already retired
- that customer demand is already validated at scale
- that pilot metrics already exist if they do not

## 3. Pilot readiness

This is the frame for:

- actual pilot launch
- live operations
- serious operating partners
- customers or sites being asked to commit

The correct claim is:

AllDone should only be described as pilot-ready once the operating loop, return policy, wash model, packaging decisions, and minimum operator software are materially real.

What must exist before this framing is used:

- locked pilot scope
- real container sourcing path
- wash and inspection SOP
- return policy
- basic operator-facing software
- clear pilot environment
- basic economic viability logic

## Practical rule

If speaking to the public:

- use the public-exposure frame

If speaking to investors:

- use the funding-readiness frame

If speaking to operators about going live:

- use the pilot-readiness frame only if the underlying proof actually exists

All three frames should still pass the release discipline in [EXPOSURE_GATE.md](EXPOSURE_GATE.md).

## Current one-line positioning

The cleanest current sentence is:

AllDone is a publicly documented, funding-ready concept and prototype for a closed-loop home provisioning system, with pilot readiness as the next proof stage rather than the current claim.
