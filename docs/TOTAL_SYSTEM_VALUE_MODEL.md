# Total System Value Model

Status: active

Purpose
-------
This document states the correct economic frame for AllDone.

AllDone should not be evaluated as a thin delivery service added on top of the current retail stack.

It should be evaluated as a replacement model for a large set of recurring costs, frictions, and waste streams that the current stack produces.

## The wrong comparison

The wrong question is:

- "Is this cheaper than sending a courier to one door?"

That framing collapses AllDone into a disposable last-mile service.

That is not the system being proposed.

## The right comparison

The right question is:

- "Is the full AllDone loop better than the full incumbent loop it replaces?"

That means comparing:

- sourcing
- packaging
- repeat purchasing behavior
- fragmented trips and deliveries
- household planning effort
- waste and overbuying
- returns / empties / disposal
- route duplication
- exception handling

against:

- reusable containers
- return logistics
- industrial washing
- refill operations
- denser recurring routes
- operator software
- loop maintenance and breakage reserves

## What the incumbent system spends today

The current stack carries costs in many places at once:

- disposable packaging materials
- packaging handling and disposal
- repeated household shopping trips
- fragmented one-off deliveries
- retail overhead tied to repetitive purchase behavior
- repeated picking, handling, and shelf replenishment
- overbuying and wasted product
- repeated route duplication across operators
- household time and planning burden
- building waste-stream clutter

Not all of those costs sit in one pocket, but they are still real system costs.

## What AllDone adds

AllDone introduces its own visible loop costs:

- container fleet capital
- wash and inspection labor
- reverse logistics
- breakage and loss reserve
- refill operations
- route execution
- container tracking and exception handling
- software for subscriptions, returns, wash, and billing

## Core thesis

The AllDone thesis is that the incumbent system is so wasteful and repetitive that a well-designed closed loop should outperform it at the system level, even after adding the visible costs of reuse, return, washing, and reverse logistics.

This is the model's real claim.

## Important discipline

Two things must still be kept separate:

1. total value created by replacing wasteful incumbent behavior
2. value actually captured by the AllDone operator

They are related, but they are not identical.

The system can be better overall before every benefit flows into operator revenue.

## What must be shown on paper

A serious AllDone model should show:

- which incumbent costs are being removed
- which new loop costs are being added
- which stakeholders benefit
- which benefits are directly monetizable
- which benefits are indirect but still strategically important

## What must eventually be measured

- route density
- labor per household served
- wash throughput and cost
- breakage and loss rate
- return compliance
- packaging turns per container
- basket stability and repeat usage

## Correct conclusion

AllDone is not trying to be slightly cheaper than the current mess at the margin.

It is trying to replace a structurally wasteful system with a denser, more standardized, and more reusable one.
