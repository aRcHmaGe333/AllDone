# Container Processing SOP

Status: approved for pilot planning

Purpose
-------
This is the first real standard operating procedure for returned containers.

It closes `P2.2`.

## Scope

Applies to:

- returned `Storage-M` containers
- returned `Dose-M` containers
- lids and removable silicone seals used in the pilot

## Process

### 1. Intake

- receive returned containers by route batch
- count units against route manifest
- record obvious missing closures or broken parts
- place returned units in dirty-intake area

### 2. Initial inspection

- segregate visibly broken glass immediately
- segregate bent or damaged lids
- segregate heavily soiled or atypical residue cases for special handling
- log quarantine reason where possible

### 3. Disassembly

- remove lids
- remove silicone gaskets where design allows
- separate glass bodies, lids, and gaskets into wash-compatible groups

### 4. Pre-rinse and wash

- pre-rinse to remove loose residue
- wash according to pilot wash chemistry and temperature standard
- ensure wash batch traceability by date and batch identifier

### 5. Drying

- dry all components fully before reassembly
- prevent standing water in closures, threads, or gasket grooves

### 6. Post-wash inspection

- inspect glass for chips, cracks, clouding, and persistent residue
- inspect lids for deformation, corrosion, and closure integrity
- inspect gaskets for tears, odor retention, deformation, and compression damage

### 7. Reassembly

- reassemble only units that pass inspection
- replace gasket if wear standard is exceeded
- send failed parts to damaged hold or retirement

### 8. Release to refill-ready inventory

- mark passing units as refill-ready
- return them to clean inventory storage by SKU

## Quarantine and retirement rules

Immediate quarantine:

- chipped rim
- crack of any size
- closure failure
- persistent contamination

Retire:

- any glass body with structural damage
- any lid that no longer seals reliably
- any gasket that fails inspection and is not worth replacing in isolation

## Records to keep

- intake count by route batch
- damaged count by failure type
- wash batch date
- gasket replacements
- refill-ready release count
