# Three Risks

Status: active

Purpose
-------
This document states the three core risks in AllDone and the proof required to reduce each one.

## 1. Demand risk

Question:

Will enough households actually adopt and keep using a closed-loop provisioning service for recurring goods?

What would reduce this risk:

- repeated interview evidence that the shopping chore is painful and frequent
- evidence that households will tolerate a scheduled return-and-pickup model
- proof that households accept the deposit and return terms
- pilot retention and repeat-order behavior

Current status:

- framed
- not yet validated with a documented interview set or live pilot data

## 2. Delivery risk

Question:

Can AllDone operate the packaging, routing, return, wash, and refill loop at acceptable cost and reliability?

What would reduce this risk:

- a locked container set
- real container sourcing quotes
- a real wash SOP
- route and stop-time measurements
- real breakage and non-return rates
- basic operator-facing software to track the loop

Current status:

- partially modeled
- not yet validated in a live operating environment

## 3. Permission risk

Question:

Can AllDone run the first pilot without getting blocked by food-safety, materials, handling, building, or local operating constraints?

What would reduce this risk:

- a regulatory and food-safety checklist for the pilot scope
- a defined wash and inspection process
- labeling and traceability decisions for the pilot
- building or site permission for the initial environment
- a clear statement of which product categories are in and out of scope for pilot one

Current status:

- acknowledged
- not yet converted into a real compliance pathway

## Minimum proof package

Before describing AllDone as financing-ready for a serious pre-seed process, the project should aim to have:

1. structured customer discovery evidence
2. one real pilot environment path
3. one real container sourcing path
4. one real wash and return process
5. one working operator flow in software
