# Fundraising Readiness

Status: active

Purpose
-------
This document converts the general pre-seed fundraising guidance into an AllDone-specific readiness checklist.

The goal is not to perform startup theater. The goal is to make AllDone legible to serious early investors and partners.

## Current position

AllDone is currently strongest as:

- a system concept
- a packaging and operating-model thesis
- a public IP-claim record
- an early prototype

This means the right fundraising posture is:

- strong enough for early investor conversations
- not mature enough to pretend the pilot is already proven

AllDone is currently weaker on:

- direct customer evidence
- live pilot proof
- financing-ready materials
- data-room discipline

## What an investor still needs to see

- a clear problem and alternative map
- a clear buyer and user definition
- a believable "why now?"
- a believable unfair advantage
- the three core risks and the proof plan for each
- the raise amount and what it buys
- a data-room index showing that diligence will not be chaotic
- live external validation that replaces current planning assumptions

## What should happen next

1. Use the new docs to create a short deck.
2. Build the actual data room against the index.
3. Start structured customer and pilot-partner conversations.
4. Replace assumptions with live evidence wherever possible.

Before any investor-facing text is sent, pass it through [EXPOSURE_GATE.md](EXPOSURE_GATE.md).

## What should not happen

- do not present the company as more mature than it is
- do not inflate TAM claims before the pilot wedge is clear
- do not try to substitute generic fundraising language for actual operating logic
