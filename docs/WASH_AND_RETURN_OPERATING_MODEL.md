# Wash And Return Operating Model

Status: approved for pilot planning

Purpose
-------
This document defines the first operating model for returns, washing, inspection, and release back into circulation.

## Chosen model

The first pilot uses:

- centralized washing
- scheduled return pickup through the delivery loop
- one depot or one wash partner as the sanitation control point

This closes `P2.1`.

## Why centralized wins for pilot one

- one inspection standard
- one sanitation standard
- one place to track container status
- lower process ambiguity
- better evidence collection for cost and breakage

Distributed washing can be revisited later if scale or geography requires it.

## Return-loop structure

Default return path:

1. household receives filled containers
2. empties are retained until the next scheduled cycle
3. empties are handed back at delivery or scheduled pickup
4. driver or operator confirms return count
5. containers are brought to depot intake
6. intake, inspection, wash, dry, reassembly, and release occur centrally

## Container state flow

- issued
- with household
- returned dirty
- inspection hold
- wash batch
- dry and reassembly
- refill-ready
- damaged hold
- retired

## Operating assumptions

- returns are primarily tied to the recurring service cycle
- one missed cycle is tolerated with reminder logic
- repeated missed returns trigger intervention
- visibly damaged containers are quarantined immediately
- gaskets are treated as replaceable wear parts

## What this model is designed to prove

- whether households comply with a simple return rhythm
- whether centralized wash is operationally manageable at pilot scale
- whether the container pool can be controlled cleanly enough for repeat refill
