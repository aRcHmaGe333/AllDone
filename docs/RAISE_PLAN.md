# Raise Plan

Status: working recommendation

Purpose
-------
This document states what a sensible early raise for AllDone should fund and what proof it should buy.

## Default recommendation

Recommended pre-seed target:

- `EUR 250,000-EUR 500,000`

Default planning target:

- `EUR 350,000`

This is not because the first pilot hard costs are that high by themselves. It is because the raise needs to cover operating runway, software hardening, pilot execution, and evidence generation over 12-18 months.

## Recommended financing posture

Default instrument:

- post-money SAFE or the closest low-friction local equivalent

Default objective:

- raise enough to prove the first narrow loop works
- avoid raising so little that the company cannot complete the proof cycle

## What the round should achieve

Within 12-18 months, the raise should aim to produce:

- one real pilot environment
- one working container loop
- a validated return and wash SOP
- a basic operator-facing software system
- initial retention and unit-economics evidence
- a cleaner basis for a larger seed round or strategic partnership discussion

## Suggested use of funds

For a `EUR 350,000` planning target:

| Category | Share | Indicative amount |
|---|---|---|
| Pilot operations and working capital | 35% | EUR 122,500 |
| Software and product hardening | 25% | EUR 87,500 |
| Container fleet, wash setup, and handling equipment | 20% | EUR 70,000 |
| Founder and key contractor runway buffer | 10% | EUR 35,000 |
| Legal, compliance, and diligence preparation | 5% | EUR 17,500 |
| Contingency | 5% | EUR 17,500 |

This should be understood as planning logic, not final budgeting.

## Ownership discipline

The raise should be structured so that:

- the round has a clear milestone narrative
- the founder understands the dilution tradeoff before signing anything
- the amount raised is tied to proof generation, not prestige

## What not to fund first

- broad geographic rollout
- heavy brand spend
- large team build-out before the loop works
- polished consumer software that hides weak operations underneath

## What the next round should be based on

The next meaningful round should be justified by evidence such as:

- active-household retention
- return compliance
- breakage and loss rates
- wash and recirculation reliability
- route density and stop efficiency
- early contribution-margin logic for the narrow wedge

Without those proof points, a larger raise would mostly fund uncertainty.
